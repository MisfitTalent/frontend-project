# GitHub Governance

Before creating a new branch, inspect the current branch list and recent history.

If an existing branch already matches the scope of the requested work, continue on that branch instead of creating a duplicate branch for the same purpose.

When duplicate branches already exist for the same workstream, move the work back to the branch whose name and purpose best match the changes.

Prefer branch names that describe one clear outcome. Avoid splitting the same UI or feature cleanup across multiple similarly named branches.

If you must create a new branch, do it only when the work is materially different from every existing branch in scope.
